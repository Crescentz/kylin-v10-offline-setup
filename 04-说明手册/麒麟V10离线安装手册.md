## 🔧 二、安装前准备

### 2.1 将文件拷贝到服务器

将 U 盘/移动硬盘插入目标服务器，把 `麒麟V10` 文件夹复制到服务器上（建议放在 `/opt/` 或 `/root/` 目录下）：

```bash
# 假设 U 盘挂载在 /mnt/usb
cp -r /mnt/usb/麒麟V10 /opt/
cd /opt/麒麟V10
```

> 💡 **如果不知道 U 盘挂载在哪里**，可以用 `lsblk` 或 `fdisk -l` 查看磁盘列表，找到你的 U 盘设备（通常是 `/dev/sdb1` 或 `/dev/sdc1`），然后：
> ```bash
> mkdir -p /mnt/usb
> mount /dev/sdb1 /mnt/usb   # 请把 sdb1 换成实际的设备名
> ```

### 2.2 检查系统是否已有部分工具

先看看目标机器上已经装了哪些工具：

```bash
# 检查 gcc
gcc --version
# 检查 make
make --version
# 检查 vim
vim --version
# 检查 curl
curl --version
# 检查 docker
docker --version
# 检查 nvidia 驱动
nvidia-smi
```

如果某个命令显示 `command not found`，说明需要安装。

> 💡 **麒麟 V10 可能已自带部分工具**，如果已有就不用重复安装。

### 2.3 创建本地 YUM 源（麒麟 V10 ISO 方式 - 推荐）

**背景**：麒麟 V10 的基础系统包（如 glibc 等系统核心库）通常通过系统 ISO 安装。以下步骤教你用 ISO 做本地源。

**步骤 1：将麒麟 V10 安装 ISO 上传到服务器**

如果你有麒麟 V10 的安装光盘 ISO 文件（如 `Kylin-Server-10-SP2-xxx.iso`），上传到服务器上，例如放到 `/root/` 目录。

**步骤 2：挂载 ISO**

```bash
mkdir -p /mnt/cdrom
mount -o loop /root/Kylin-Server-10-SP2-xxx.iso /mnt/cdrom
```

**步骤 3：配置本地 YUM 源**

```bash
# 备份原有 repo 配置
mkdir -p /etc/yum.repos.d/backup
mv /etc/yum.repos.d/*.repo /etc/yum.repos.d/backup/ 2>/dev/null

# 创建本地源配置
cat > /etc/yum.repos.d/local.repo << 'EOF'
[local-baseos]
name=Kylin V10 Local BaseOS
baseurl=file:///mnt/cdrom/BaseOS
gpgcheck=0
enabled=1

[local-appstream]
name=Kylin V10 Local AppStream
baseurl=file:///mnt/cdrom/AppStream
gpgcheck=0
enabled=1
EOF

# 清理缓存并刷新
yum clean all
yum makecache
```

> 💡 **没有麒麟 ISO？** 如果找不到麒麟 V10 的 ISO，我们的 RPM 包（03-开发工具/）也可以直接使用，见下一节。

## 🐳 四、安装 Docker + Docker Compose

### 4.1 安装 Docker 引擎

Docker 使用静态二进制安装方式，不需要依赖 RPM 包，最简单可靠。

```bash
# 进入 Docker 目录
cd /opt/麒麟V10/02-Docker/

# 解压 Docker 二进制包
tar -xzvf docker-29.5.2.tgz

# 将 Docker 程序复制到系统目录
cp docker/* /usr/bin/

# 创建 systemd 服务文件
cat > /etc/systemd/system/docker.service << 'EOF'
[Unit]
Description=Docker Application Container Engine
Documentation=https://docs.docker.com
After=network-online.target firewalld.service
Wants=network-online.target

[Service]
Type=notify
ExecStart=/usr/bin/dockerd
ExecReload=/bin/kill -s HUP $MAINPID
LimitNOFILE=infinity
LimitNPROC=infinity
LimitCORE=infinity
TimeoutStartSec=0
Delegate=yes
KillMode=process
Restart=on-failure
StartLimitBurst=3
StartLimitInterval=60s

[Install]
WantedBy=multi-user.target
EOF

# 重新加载 systemd 配置
systemctl daemon-reload

# 启动 Docker
systemctl start docker

# 设置 Docker 开机自启
systemctl enable docker
```

### 4.2 安装 Docker Compose

```bash
# 复制 docker-compose 到系统目录
cp /opt/麒麟V10/02-Docker/docker-compose-linux-x86_64 /usr/local/bin/docker-compose

# 添加可执行权限
chmod +x /usr/local/bin/docker-compose
```

### 4.3 验证安装

```bash
# 检查 Docker
docker --version
# 输出示例：Docker version 29.5.2, build xxx

# 检查 Docker Compose
docker-compose --version
# 输出示例：Docker Compose version v5.1.4

# 检查 Docker 运行状态
systemctl status docker
# 输出应为 active (running)
```

都正常显示就算安装成功 ✅

## 🎮 五、安装 CUDA 11.8 + NVIDIA 驱动（关键部分）

### 5.1 先搞清楚：CUDA 和 NVIDIA 驱动的关系

```
CUDA 安装文件 (cuda_11.8.0_520.61.05_linux.run)
├── 📦 CUDA Toolkit      (CUDA 工具包 - 必须安装)
│   ├── nvcc 编译器
│   ├── CUDA 库文件
│   └── 开发头文件
│
└── 🎮 NVIDIA 驱动       (显卡驱动 - 可选)
    └── 版本 520.61.05（支持 RTX 4090）
```

**结论：** CUDA 安装文件 = CUDA 工具包 + NVIDIA 驱动，**一个文件包含了两个东西**。

> ⚠️  本安装包中的 `cuda_11.8.0_520.61.05_linux.run` **已经下载完成**（3.2GB），不需要再下载其他文件！

### 5.2 关于 RTX 4090 和驱动版本的说明

| 驱动版本 | 发布时间 | 是否支持 RTX 4090 | 说明 |
|----------|---------|-------------------|------|
| 520.61.05（CUDA自带） | 2022年11月 | ✅ **支持** | RTX 4090 2022年10月发布，此驱动可用 |
| 545+ / 550+ | 2023-2024年 | ✅ 支持 | 新驱动性能更好，但需要单独下载 |
| 570+ / 580+ / 595+ | 2025-2026年 | ✅ 支持 | 最新驱动，功能最全 |

**建议：** CUDA 11.8 自带的 520.61.05 驱动可以驱动 RTX 4090，但不是最优选择。我们提供了两种安装方案（见下方）。

### 5.3 安装前的准备

```bash
# 1. 确认 gcc 已安装（CUDA 安装需要）
gcc --version

# 2. 查看当前内核版本
uname -r
# 输出类似: 4.19.90-89.11.v2401.ky10.x86_64

# 3. 如果有麒麟 V10 ISO，挂载并安装 kernel-devel（匹配内核版本的开发包）
#    这是可选的，只有安装NVIDIA驱动时需要
# mount -o loop /path/to/Kylin-ISO.iso /mnt/cdrom
# yum install -y kernel-devel
```

### 5.4 方案 A：安装 CUDA 工具包 + 自带驱动（最简单，推荐新手）

这种方案最简单，一个命令解决所有问题。驱动 520.61.05 可以驱动 RTX 4090。

```bash
# 进入目录
cd /opt/麒麟V10/01-CUDA/

# 给文件加执行权限
chmod +x cuda_11.8.0_520.61.05_linux.run

# 运行安装程序（交互式）
sudo sh cuda_11.8.0_520.61.05_linux.run
```

**交互安装步骤：**

| 步骤 | 画面 | 操作 |
|------|------|------|
| 1 | License 协议 | 按 `q` 跳过阅读，输入 `accept` 回车 |
| 2 | 组件选择 | **Driver 默认已勾选**，如果你也想装驱动，保持不动；如果只想装 CUDA，用方向键移动到 Driver 按空格取消勾选 |
| 3 | 开始安装 | 移动到 `Install` 按回车 |
| 4 | 等待 | 安装过程约 1-3 分钟 |

**静默安装（非交互，适合脚本）：**

```bash
# 只安装 CUDA 工具包（不安装驱动）
chmod +x cuda_11.8.0_520.61.05_linux.run
sudo sh cuda_11.8.0_520.61.05_linux.run --silent --toolkit

# 安装 CUDA 工具包 + 驱动（全部安装）
sudo sh cuda_11.8.0_520.61.05_linux.run --silent
```

### 5.5 方案 B：只安装 CUDA 工具包，另外装新版驱动

# sudo sh cuda_11.8.0_520.61.05_linux.run --silent --toolkit

**方法一：从 NVIDIA 官网下载**

打开浏览器访问：https://www.nvidia.cn/geforce/drivers/

- 产品型号：GeForce RTX 4090
- 操作系统：Linux 64-bit

## ❓ 八、常见问题 (FAQ)

### Q1：安装 RPM 时提示 "key ID 错误" 或 "签名验证失败"

**解决**：安装时加上 `--nogpgcheck` 参数：
```bash
rpm -ivh *.rpm --force --nodeps --nogpgcheck
```

### Q2：Docker 启动失败（报错 "Failed to start Docker"）

**解决步骤**：
```bash
# 查看 Docker 日志定位原因
journalctl -u docker.service -n 50

# 常见原因：iptables 规则冲突或 cgroup 驱动不匹配
# 可以尝试临时禁用防火墙测试：
# systemctl stop firewalld
# systemctl start docker

# 如果仍然失败，查看错误详情：
dockerd --debug 2>&1 | head -20
```

> ⚠️ **注意**：不到万不得已不要删除 `/var/lib/docker`，否则会丢失所有镜像和容器数据。

### Q3：CUDA 安装时报错 "Failed to verify gcc version"

**解决**：CUDA 11.8 支持 gcc 最高到 10.x，我们的 gcc 8.5 完全兼容。如果还是报错，加上 `--override` 参数：
```bash
sudo sh cuda_11.8.0_520.61.05_linux.run --override
```

### Q4：nvidia-smi 报错 "No devices were found"

**可能原因**：
1. 显卡没有正确安装（检查物理连接）
2. 驱动没有正确安装（重装驱动）
3. 需要先屏蔽 nouveau 驱动：
   ```bash
   # 检查是否使用了 nouveau
   lsmod | grep nouveau
   
   # 如果显示有输出，需要屏蔽
   echo "blacklist nouveau" >> /etc/modprobe.d/blacklist.conf
   dracut -f
   reboot
   ```

### Q5：安装时提示 "kernel-devel not found"

**解决**：麒麟 V10 的内核版本为 4.19.90，需要安装对应的 kernel-devel 包：
```bash
# 方法1：通过麒麟 V10 ISO 安装（推荐）
# 挂载 ISO 后：
yum install -y kernel-devel

# 方法2：检查当前内核版本并下载对应包
uname -r
# 输出示例：4.19.90-89.11.v2401.ky10.x86_64
```

### Q6：yum 命令报错 "Could not resolve host"

**正常情况**，因为服务器没有联网。我们已经通过 RPM 包提前下载好了所有需要的软件：
- 开发工具 → 用 `rpm -ivh` 命令安装
- Docker → 用二进制方式安装
- CUDA → 用 runfile 安装

## 🔧 十、关于内核升级的重要说明

### 10.1 `05-内核升级` 目录的作用

如果你需要升级麒麟 V10 的内核，可以将内核 RPM 包放在 `05-内核升级/` 目录下，方便管理和携带。

**当前已包含的文件：**
| 文件 | 说明 | 升级版本 |
|------|------|---------|
| `kernel.tar.gz` (241MB) | 包含 kernel、kernel-core、kernel-modules 等RPM包 | 4.19.90-89.28.v2401.ky10 |
| `内核升级命令.txt` | 升级操作命令 | 解压→安装→重启 |

> 当前系统内核版本：4.19.90-89.11.v2401.ky10  
> 升级目标版本：4.19.90-89.28.v2401.ky10

**这个目录只是一个存放位置**，不会影响其他任何文件。

### 10.2 升级内核后对现有软件的影响

| 软件 | 升级内核后是否受影响 | 说明 |
|------|---------------------|------|
| **gcc / g++ / make / vim / curl** | ✅ **不影响** | 这些是用户态工具，不依赖内核版本 |
| **Docker** | ✅ **不影响** | Docker 重启后即可正常工作 |
| **CUDA Toolkit** | ✅ **不影响** | CUDA 使用用户态库，不依赖内核 |
| **NVIDIA 驱动** | ⚠️ **需要重装！** | 驱动内核模块需要针对新内核重新编译 |
| **kernel-devel** | ⚠️ **需要同步升级** | 必须匹配新内核版本 |

**最重要的影响：升级内核后，NVIDIA 驱动会失效**，因为驱动编译的内核模块绑定在旧内核上。所以：

```
升级内核流程：
1. 下载新内核 RPM 包 → 放到 05-内核升级/
2. 安装新内核 → rpm -ivh kernel-*.rpm
3. 重启 → select new kernel
4. 重新安装 NVIDIA 驱动 → sh NVIDIA-Linux-*.run
5. Docker 重启 → systemctl restart docker
```

### 10.3 如果暂时不需要升级内核

你可以直接忽略 `05-内核升级` 目录，完全不影响其他软件的安装。
